# Руководство администратора

Актуализировано: 23.05.2026.

Администратор управляет пользователями, учебными справочниками, назначениями преподавателей, отчетами и аудитом.

## Вход

Откройте:

```text
http://localhost:3000/login
```

Демо-доступ:

```text
admin / admin123
```

Для демонстрационного стенда можно наполнить систему расширенным набором:

```bash
docker compose exec backend python manage.py create_demo_data --reset --full
```

Будут созданы дополнительные преподаватели `demo_teacher_01` ... `demo_teacher_05` и студенты `demo_student_001` ... `demo_student_059`.

После входа доступны разделы:

- `Обзор`;
- `Администратор`;
- `Отчеты`;
- `Уведомления`.

## Кабинет администратора

Маршрут frontend:

```text
/admin
```

Рабочая область разделена на вкладки:

- пользователи;
- учебный процесс;
- аудит.

В верхней части кабинета отображаются сводные показатели: пользователи, группы, оценки, студенты, преподаватели, дисциплины и назначения.

## Пользователи

Администратор может:

- создать пользователя;
- изменить пользователя;
- сбросить пароль;
- отключить или включить пользователя;
- задать срок действия доступа;
- выбрать роль.

Роли:

| Роль | Назначение |
| --- | --- |
| `admin` | полный доступ к администрированию |
| `teacher` | кабинет преподавателя, журнал, отчеты по своим назначениям |
| `student` | кабинет студента и личные данные |

При выборе роли в форме появляются дополнительные поля:

- `Студент`:
  - номер студента;
  - группа;
  - дата зачисления.
- `Преподаватель`:
  - табельный номер;
  - должность.

Если для преподавателя оставить табельный номер пустым, система назначит номер автоматически в формате `T-000001`.

Backend API:

```text
/api/users/
POST /api/users/{id}/reset-password/
```

## Учебный процесс

Во вкладке `Учебный процесс` администратор управляет:

- группами;
- дисциплинами;
- учебными периодами;
- назначениями преподавателей.

Перед началом работы журнала должны существовать:

1. группа;
2. дисциплина;
3. учебный период;
4. преподаватель с профилем;
5. студент с профилем и группой;
6. назначение преподавателя на дисциплину, группу и период.

Backend API:

```text
/api/education/groups/
/api/education/subjects/
/api/education/periods/
/api/education/students/
/api/education/teachers/
/api/education/teaching-assignments/
```

## Назначения преподавателей

Назначение связывает:

- преподавателя;
- дисциплину;
- группу;
- учебный период.

Именно назначение используется журналом оценок, занятиями, посещаемостью и отчетами. Если преподаватель не отображается в списке назначений, проверьте, что у пользователя роль `teacher` и создан профиль преподавателя.

## Отчеты

Раздел frontend:

```text
/reports
```

Администратор может формировать отчеты по всем данным.

Типы:

- успеваемость;
- посещаемость.

Форматы:

- `csv`;
- `xlsx`;
- `pdf`.

Backend API:

```text
POST /api/reports/requests/generate/
GET /api/reports/requests/
```

## Уведомления

Раздел frontend:

```text
/notifications
```

Уведомления привязаны к конкретному пользователю. Если администратор отмечает свое уведомление прочитанным, это не влияет на уведомления преподавателя.

Backend API:

```text
GET /api/notifications/notifications/
POST /api/notifications/notifications/{id}/mark-read/
```

## Аудит

Во вкладке `Аудит` отображаются события системы:

- вход;
- неудачный вход;
- выход;
- создание и изменение пользователей;
- изменения оценок;
- изменения посещаемости;
- создание отчетов.

В аудите полезны поля:

- дата;
- пользователь;
- действие;
- тип объекта;
- объект;
- request id;
- IP-адрес;
- user-agent.

Backend API:

```text
GET /api/audit/logs/
```
