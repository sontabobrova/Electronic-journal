# Руководство администратора

## Основные обязанности

Администратор ведет пользователей, учебные справочники, назначения преподавателей и контролирует журнал аудита.

## Пользователи

Пользователи доступны через `/api/users/`.

Роли:

- `admin` - полный доступ к управлению системой.
- `teacher` - доступ к своим назначениям, журналам и отчетам.
- `student` - доступ только к личным оценкам и посещаемости.

Для сброса пароля используется:

```text
POST /api/users/{id}/reset-password/
```

## Учебные справочники

Справочники доступны в `/api/education/`:

- группы;
- дисциплины;
- учебные периоды;
- профили студентов;
- профили преподавателей;
- назначения преподавателей.

Перед началом работы журнала должны быть созданы группа, дисциплина, период, профиль преподавателя, профиль студента и назначение.

## Отчеты

Отчеты формируются через:

```text
POST /api/reports/requests/generate/
```

Типы отчетов:

- `grades`
- `attendance`

Форматы:

- `csv`
- `xlsx`
- `pdf`

## Аудит

Журнал аудита доступен только администратору:

```text
GET /api/audit/logs/
```

При разборе инцидентов используйте `request_id`, IP-адрес, пользователя, действие и объект изменения.
