# Руководство пользователя

Актуализировано: 22.05.2026.

Документ описывает работу студента и преподавателя в интерфейсе электронного журнала.

## Вход и выход

Откройте:

```text
http://localhost:3000/login
```

Введите логин и пароль. После входа система откроет доступные разделы в зависимости от роли.

Демо-учетные записи:

| Роль | Логин | Пароль |
| --- | --- | --- |
| Преподаватель | `teacher` | `teacher123` |
| Студент | `student` | `student123` |

Для выхода нажмите кнопку выхода в верхней панели.

## Обзор

Маршрут:

```text
/
```

Страница `Обзор` показывает доступные пользователю рабочие разделы и быстрые переходы. Техническая информация о проекте на этой странице не отображается.

## Студент

Маршрут:

```text
/student
```

Студент видит только свои данные.

Доступно:

- профиль студента;
- группа;
- средний балл;
- сводка посещаемости;
- список оценок;
- список посещений.

Студент не может:

- создавать или менять оценки;
- создавать занятия;
- отмечать посещаемость;
- видеть данные других студентов;
- работать с отчетами;
- работать с административным разделом.

Backend API личного кабинета:

```text
GET /api/journal/student/dashboard/
GET /api/journal/student/grades/
GET /api/journal/student/attendance/
```

## Преподаватель

Маршрут:

```text
/teacher
```

Преподаватель работает только с назначенными ему группами, дисциплинами и периодами.

Доступно:

- профиль преподавателя;
- сводка по назначениям;
- список назначений;
- студенты назначенных групп;
- работы журнала;
- оценки;
- занятия;
- посещаемость.

Преподаватель может:

- создавать, изменять и удалять работы журнала;
- выставлять, изменять и удалять оценки;
- создавать, изменять и удалять занятия;
- отмечать, изменять и удалять посещаемость;
- формировать отчеты по своим данным;
- читать свои уведомления.

Преподаватель не может:

- управлять пользователями;
- управлять учебными справочниками;
- видеть чужие назначения;
- менять оценки и посещаемость по чужим группам.

Backend API кабинета преподавателя:

```text
GET /api/journal/teacher/dashboard/
GET /api/journal/teacher/assignments/
GET /api/journal/teacher/students/
GET /api/journal/teacher/grade-works/
GET /api/journal/teacher/grades/
GET /api/journal/teacher/class-sessions/
GET /api/journal/teacher/attendance/
```

## Отчеты преподавателя

Маршрут:

```text
/reports
```

Преподаватель может формировать отчеты:

- по успеваемости;
- по посещаемости.

Данные ограничены назначениями текущего преподавателя.

Форматы:

- `csv`;
- `xlsx`;
- `pdf`.

## Уведомления преподавателя

Маршрут:

```text
/notifications
```

Преподаватель получает персональные уведомления. Сейчас используется напоминание о закрытии успеваемости за 3 дня до окончания активного учебного периода.

Отметка `прочитано` применяется только к уведомлению текущего пользователя.

## API-авторизация

При работе напрямую с API:

```text
POST /api/auth/login/
POST /api/auth/logout/
GET /api/auth/me/
```

После входа используйте токен:

```text
Authorization: Token <token>
```
